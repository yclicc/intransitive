# Intransitive Dice
A naive implementation of a Dice class for non-transitive dice with an illustration of the fact that Grimes' dice (described [here](https://singingbanana.com/dice/article.htm)) beat each other, but that this reverses when two dice of the same kind are added.

![Grimes' dice (I call the olive die green)](https://singingbanana.com/dice/rowlettdicelabelled.png)

We further see that with three dice of the same kind added together it becomes possible (or even likely) that there will be a draw with three dice of a different colour. This pattern repeats (though I haven't proven this for all N), with the probabilities getting closer to 0.5 for more and more dice, with draws only possible with 0 mod 3 dice. This is because the red die has faces with values that are 0 mod 3, the blue dice has faces that are 2 mod 3 and the green dice has faces that are 1 mod 3. Consequently, the only time the sum of such numbers can possibly be equal is when 0 mod 3 (e.g. 3, 6, 9, 12) of them are added together.

Additionally, the green dice seems to be marginally "the best" in that when it ought to win it wins with a higher probability and when it ought to lose it doesn't lose as badly as the other two (which are "equally good/bad" as each other).

Exercise (post as an issue if you have the answer before anyone else): Prove that the order always flips between 1 mod 3 and 2 mod 3 dice.
